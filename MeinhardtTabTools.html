Meinhardt Tab Tools Description

Meinhardt tab screenshot (top reference).

Meinhardt Tab is a curated collection of tools specially created for modeling MEP projects and formatting sheets and files to Meinhardt standards. It centralizes day-to-day production commands for setup, coordination, data quality, and drawing package consistency.

Scope: All tools under Meinhardt tab except MEP Lab panel. This page is browser-formatted for easy onboarding and includes UI screenshots.

Installation Instructions (Header Summary)
  1. Install pyRevit and verify the pyRevit ribbon appears in Revit.
  2. Copy MHTTools.extension into %APPDATA%\\pyRevit\\Extensions.
  3. Reload pyRevit (or restart Revit), then confirm the MEINHARDT tab is visible.

User Instructions: Start by selecting the panel from the table of contents below, open the tool card, then follow the basic operation steps. For tools with screenshots, match your UI to the sample image to confirm correct workflow.

Installation Guide (Simple Setup)

Quick setup for new users. Keep it simple: install pyRevit, copy one folder, reload, then check one tool.

1) Install pyRevit

  • Install pyRevit and open Revit once.
  • Confirm the pyRevit tab appears in the ribbon.

2) Copy the Meinhardt Extension

  • Copy MHTTools.extension into %APPDATA%\\pyRevit\\Extensions.
  • Do not copy only MEINHARDT.tab. Copy the full MHTTools.extension folder.

3) Reload and Verify

  • Click pyRevit Reload (or restart Revit).
  • Confirm the MEINHARDT tab is visible.
  • Run Search Tool or ToolsDescription as a quick check.

Quick Troubleshooting

  • If MEINHARDT tab is missing, check folder name is exactly MHTTools.extension.
  • Reload pyRevit again after confirming folder location.

Creates coordinated rooms, spaces, areas, and zones from host or linked context with boundary support.

08 Selection Filter

Recreation of JoTools Selection Filter for teams that no longer have access to JoTools Free Revit Tools in Revit 2015 environments. This tool is still in progress, but it is already usable for production filtering tasks.

How it works: Expand a parameter, tick one or more values, choose mode (Keep Checked or Unselect Checked), then apply to refine the current set.

How it differs from default Revit Selection Filter: Revit's built-in filter mostly narrows by category, while this tool filters by parameter values within and across categories, giving much finer control for QA and targeted edits.

  • Set scope (for example Current Selection).
  • Use Search to quickly find parameter names.
  • Expand parameter values and check match criteria.
  • Choose Keep or Unselect mode and click Apply Filter.

Actual UI screenshot: GM Selection Filter with scope selector, parameter/value tree, and Keep or Unselect filtering mode.

Views Links Panel

General description: Sheet-driven link management for placed views. Use it to batch control link display mode, halftone, hide or unhide link types, and apply linked-grid visibility actions when the active Revit API supports that category override path.

Plan Link Setup

Focused setup tool for the repeated mechanical-plan workflow: select sheets and views, choose one loaded source link, choose the linked plan that already shows rooms, then let the tool finish the override sequence automatically.

  • Use the structured sheet tree to choose one or many sheets, then confirm which placed views should be edited.
  • Select the room-tag source link and the linked view to reuse.
  • The tool hides linked grids on all visible links, applies By Linked View to the selected link, then freezes that link to Custom with View Filters set to By Host.

Version 2 keeps the same selection-panel logic as Views Links Manager but reduces the settings surface to the exact manual Revit sequence needed for room-tag coordination plans.

Auto Room Names

Single-click current-view tool that tries to find the correct architectural linked plan automatically before applying the room-name setup.

  • Scans visible architectural links in the active view only.
  • Scores linked views using host level match, view type, name similarity, and actual room-tag presence in the linked view.
  • Applies the best match, hides linked grids on all visible links, and finishes the selected source link with the same Custom cleanup used by Plan Link Setup.

Use this first for the common case. Keep Plan Link Setup for projects where naming is inconsistent or when you need to override the automatic choice.

QUICK FORMAT

Smart link configuration tool that intelligently finds and displays room names from linked plan views while hiding grids and dimensions.

  • Scans all visible links in the active view.
  • Releases RVT Links from view template control if needed.
  • For each link, automatically finds the best plan view at the matching level that shows room names.
  • Sets each link to Custom mode with the selected room view and View Filters set to By Host.
  • Hides grids and dimensions on all links for clean display.

Intelligent room name coordination - finds the right view automatically and configures links perfectly.

Hide Grids

Standalone tool to hide grids on all visible linked models in the current view.

  • Scans all visible links in the active view.
  • Hides the grid category on each link's visibility settings.
  • Applies changes immediately without additional configuration.

Simple grid hiding for linked models - use when you only need to hide grids without other link formatting.

Views Links Manager

Three-panel batch editor for Revit links across selected sheets and their placed views.

  • Select sheets in the center tree and confirm which placed views should be edited.
  • Select target links, or use the architectural-link shortcut to isolate the main architectural model quickly.
  • Apply By Host View or By Linked View settings, halftone, hide or unhide actions, and linked-grid actions in one batch.

Actual UI screenshot: Views Links Manager with sheet selection, link controls, and batch action panels visible in one workspace.

MHT SHEETS Panel

General description: High-impact sheet production toolkit for Meinhardt deliverables. Use these tools to standardize titleblock data, automate naming logic, align viewports consistently, generate sheets, and prepare issue-ready drawing sets faster.

FORMAT

Main formatting command for production sheets. It combines target selection, sheet filtering, parameter editing, smart auto fill, and smart viewport alignment in one workspace.

  • Tick target sheets from the center tree.
  • Select Sheet, Title Block, or Placed View parameter mode.
  • Apply direct edits, Smart Auto Fill, Smart Align, and Auto Revision update as needed.

Actual UI screenshot: FORMAT V3 full panel interface with sheet tree, target mode switch, and parameter editor.

SheetGenerator

Generates planned and dependent views, previews output, and creates sheets from Excel-driven definitions.

  • Load Excel source and choose worksheet/titleblock.
  • Preview generated views/sheets.
  • Run Generate Views then Generate Sheets.

Actual UI screenshot: MHT Sheet Generator panel with source controls, planned/generated view list, and preview pane.

RENUMBER_RENAME

Batch renames sheet numbers and sheet names with separate find, replace, prefix, and suffix controls.

  • Use the current Project Browser selection or choose sheets from the built-in selector when nothing is preselected.
  • Set independent Find, Replace, Prefix, and Suffix values for both sheet numbers and sheet names.
  • Run Rename and review the updated sheet browser entries after the tool refreshes the browser.

AlignViewports

Aligns viewports across selected sheets while optionally normalizing crop settings, legends, overlap behavior, and titleblocks.

  • Select the target sheets and choose the main sheet that will act as the alignment reference.
  • Enable optional behaviors such as Apply same CropBox/ScopeBox, Overlap multiple ViewPlans, Align Legend, or Apply same TitleBlock.
  • Run Align Viewports and verify that sheets follow the same placement standard as the reference sheet.

Mechanical Services Panel

General description: Mechanical QA tools focused on duct model health, parameter completeness, and fast issue isolation.

SMART CHECKER

Three-panel duct QA workspace for scope selection, editable rule authoring, and result review with automatic navigation to failing ducts.

  • Choose the duct scope and optional level filter, then preview how many ducts are in the check set.
  • Edit rule names, parameter aliases, rule mode, allowed values, or minimum thresholds in the middle panel and enable only the checks you want to run.
  • Run Check, inspect the failing results list, and click any row to jump directly to that duct in the active or 3D view.

MEP Create Panel

General description: Creation and generation workflows for systems, views, worksets, fittings, and dimensions.

BatchCreateSystems

Creates new MEP system types from an Excel-style import template, with export support for existing system type data.

  • Export the current system type names, abbreviations, and classifications into the workbook template.
  • Fill the import sheet with the new system type names, abbreviations, and base system types to duplicate.
  • Run Import to duplicate the base system types and apply the configured abbreviations in one batch.

BatchDependentViewCreation

Creates dependent views from selected parent views and scope boxes.

BatchWorksetCreation

Creates multiple worksets from a small list editor or from an imported text file in a workshared model.

  • Load workset names from a text file or add rows manually in the grid.
  • Use the add and remove actions to refine the workset list before creation.
  • Click the confirm action to create all listed worksets in a single batch transaction.

PipeTypeFromCSV

Creates pipe schedule/segment sizes from CSV input.

CreateSection

Creates section views along selected linear elements.

QuickDimension

Creates one dimension line to multiple selected references.

Transition

Creates transition fitting between selected open connectors.

MEP Data Panel

General description: Parameter/data transfer, linked-model data projection, level control, and material database operations.

ElementChangeLevel

Changes the reference level of selected MEP elements while preserving the intended elevation or offset behavior for supported categories.

  • Choose the new level from the level list or pick a reference object to derive its associated level.
  • Run the level change against the current selection without manually rehosting each element.
  • Use it for ducts, pipes, fittings, accessories, and spaces when the level needs to change but the modeled position must stay consistent.

ElevationUnder

Measures the distance from selected elements up to the nearest floor or roof above and stores the result in a chosen parameter.

  • Confirm or change the target storage parameter name.
  • Select the 3D view used for reference intersection if the active view is not suitable.
  • Run Compute selected to write the calculated elevation-under value back onto the selected elements.

Linked Room Parameter

Transfers room data from a linked model into selected host elements using a step-by-step mapping workflow.

  • Choose the linked model and detect or manually pick source rooms in the current view.
  • Select target categories and host elements, then review the auto-generated parameter mappings.
  • Adjust strictness, skip or append behavior, and run Transfer Parameters.

Linked Elements Parameter

Transfers parameter values from linked source elements into selected host elements with category filtering and mapping controls.

  • Choose a linked model, source element type, and detection mode or manual pick workflow.
  • Select target categories and host elements, then review and override the proposed mappings.
  • Set auto-map strictness and transfer options, then execute the batch transfer.

Parameter Transfer

Copies or moves values from one parameter to another on the currently selected elements.

  • Select the source parameter and compatible destination parameter.
  • Optionally convert values to string or clear the source after transfer.
  • Create a missing destination parameter when needed, then apply the transfer.

ExportMaterialsGraphics

Exports material graphics data to JSON for database workflows.

UpdateMaterials

Updates/regenerates materials database and applies graphics/assets.

RoomToSpace

Copies room values from another open document or linked source into spaces in the current project using line-by-line field mapping.

  • Select the source room document and the target space document.
  • Define source room parameter expressions and separators on the left, then map each line to a single target space parameter on the right.
  • Run the transfer to populate spaces from rooms located at matching space points.
